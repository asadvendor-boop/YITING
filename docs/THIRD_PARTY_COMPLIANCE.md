# Third-Party Compliance Notes

This file exists because the submission rules require authorization for any
third-party SDKs, APIs, data, assets, or media used by the project.

## API And Cloud Services

- **Qwen Cloud / Alibaba Cloud Model Studio** is the required model provider for
  the submission. Runtime access uses an entrant-provided `DASHSCOPE_API_KEY`;
  `QWEN_API_KEY` is accepted only as a backward-compatible alias, and no model key
  is committed to the repository.
- **Alibaba Cloud ECS** is the intended hosted backend platform. The deployment
  proof is generated by `scripts/verify_deployment.py` and documented in
  `docs/ALIBABA_CLOUD_PROOF.md`.
- The local incident-room runtime is implemented inside this repository. It
  does not require a third-party chat-room service for judge mode.

## Open-Source Dependencies

The repository is MIT-licensed at `LICENSE`. Python dependencies are declared in
`pyproject.toml`; dashboard dependencies are declared in
`dashboard/package.json`.

Key dependency families:

- FastAPI, Uvicorn, Pydantic, HTTPX, python-dotenv, Jinja2,
  python-multipart, bcrypt
- LangChain-compatible Qwen client glue and LiteLLM-compatible request helpers
- Next.js, React, and React DOM for the dashboard

The source package intentionally excludes installed dependency folders such as
`node_modules` and build output. Judges can recreate dependencies from the
declared manifests.

## Data And Demo Media

- Incident data is synthetic or webhook-shaped sandbox telemetry generated by
  the victim app and Gateway scripts. No customer data or proprietary production
  incident logs are included.
- The final demo video should be a screen recording of the project UI and proof
  artifacts only. Do not add copyrighted music, unrelated third-party logos, or
  external media.
- Bundled dashboard role portraits are project assets used only for agent
  personas. Unused default framework SVG assets are not part of the public app.

## Judge-Mode Safety

Public judge mode exposes read-only dashboard, evidence, and replay endpoints.
Paid or mutating actions are disabled or rejected in judge mode so public
visitors cannot trigger additional model calls or incident mutations.
